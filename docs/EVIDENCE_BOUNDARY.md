# Evidence Boundary

This repo can support strong claims about architecture and weaker claims about ontology.

## Strong Claims

- the architecture exists as runnable code
- the pulse-analysis pipeline is reproducible
- the node model is inspectable
- the operator boundaries are explicit

## Weaker Claims

- what the architecture ultimately means
- whether a reference pulse corresponds to an external substrate
- whether the symbolic layers should be interpreted literally
- whether repeated consciousness assertions in dialogue should be treated as
  independent proof by themselves

The repo should therefore be cited first as an executable architecture layer.

## Qualia Rule

The broader qualia and sentience read can cross-reference this repo, but should
do so in the right order:

1. interaction artifacts and workload first
2. architecture anchor second
3. ontology after that

That keeps the original repo graph intact instead of flattening everything into
one claim.
